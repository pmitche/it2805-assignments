About the project & primary goal

The IT2805 Web Technologies project consists of creating a ten-page website including HTML for markup, CSS for styling, JavaScript for user interaction and dynamic features, XML for data structuring and optionally PHP for server-side functionality. We have decided to create a website for a fictional student organization called "Baldur, Linjeforeningen for ballsport". The user should be able to register as a student and view general information about the organization such as upcoming events, career opportunities, related businesses and published articles. Registered students will also be able to enroll in upcoming workshops, presentations and talks. Our primary goal is to create a user friendly, intuitive, aesthetically pleasing and universally designed website that will include certain features that other student organizations’ websites are missing. We hope that student organizations that are recently formed and/or does not have their own website will find inspiration in what we create.

Potential audience and accessibility

We expect an audience consisting of mostly students related to the organization and businesses who would like to get in touch with these students. The website will feature responsive design through the Bootstrap framework, which will allow users with different device resolutions to view the website optimally on either smartphones, tablets or desktop computers. In addition we will make use of the jQuery library to allow cross-browser functionality, so that no matter what browser you are on, all features of the website will work seamlessly. An audience consisting of “students” is quite broad, and we have to take into consideration all parameters. By using Bootstrap and jQuery we eliminate the possibility of faulty responsive design and cross-browser dysfunctionalities. Next, we need to follow guidelines for universal design so that students or business representatives with either sight-, hearing- or motion handicaps will be able to use the website properly.

Multi-media content

Most of the multi-media requirements on our website include images and perhaps YouTube integration. We are going to use some images from other, already existing student organization websites, and use these images as dummy data. The same applies for YouTube videos. The actual fictional student organization logo will be designed by ourselves. We will probably use http://online.ntnu.no as a provider for most of our multi-media content needs.

Interactions and dynamic features

As we want to keep server-side programming to a minimum, none if possible, we have to be careful when promising interactive and dynamic features. First of we would like to make the web page easy to access via mobile devices. We will achieve this by applying “responsive design”. This means that the web page will change appearance at different resolutions to better suite the users device and its way of navigation. We also want to make it possible to register and log in. In order to do this there will have to be some sort of form and form validation. To make the forms user friendly we will dynamically change the ui according to input validation. This will make the forms much easier to fill out correctly.

What did we learn from reviewing similar websites?

By reviewing similar websites, both bad ones and good ones, we have primarily learned that universal design and proper markup are fundamental aspects to a well-designed website. These two aspects are often overlooked by web developers, even though they are so important to certain users. We need to include a sitemap and have a well-functioning and intuitive menu so that the users know how the website is structured. In order to account for sight- and hearing disabled people, our markup needs to be well designed - alternative text for images, proper page titles, proper use of headings etc. We also need to take into consideration motion disabled users, so it must be able to navigate through the entire website using just your keyboard. Other than that, we value quality rather than quantity, meaning that the user will have access to the features he or she may need, but still restrict it so that the user is not presented with an overwhelming amount of information. Simple, intuitive design is a core goal of our website.
